Release Notes - May - 1 2017

Features

Product Groups

The Product Groups feature allows to group together multiple abstract products for various reasons. One of the most common use cases of this feature is to group same products with different colors together to make user’s browsing experience more convenient.

Once products are grouped together, on catalog and product detail pages instead of displaying individual product you can display the group. This means with each product you can also display references to other products which are in the same group as your product.

Migration Guides

To upgrade, follow the steps described below:

Apply every minor and patch:

composer update "spryker/*"

Once that is done, upgrade to the new Product Group major and its dependencies:

composer require spryker/product-group:"^1.0.0" spryker/product-group-collector:"^1.0.0" spryker/product-management:"^0.4.0"

Improvements

CodeStyle Checker

So far the code style checker did only work for project level. It did not resolve the folder names of vendor namespaced eco system modules and alike. With this release we introduce an improvement. The sniffer now formats module names in split and non-split vendor folder differently and checks for both.

PropelQueryBuilder Filtering and PropelOrm Formatting

With this release we introduce more improvements for Propel. We added support for sorting, pagination and column selection. In addition we also added AssociativeArrayFormatter class to always return results as associative arrays no matter how many columns were selected.

Bugfixes

Salutation in Customer Data

Previously we had an issue that salutation was persisted in the customer data and was available in customer view, yet it was not shown in the backend. This issue is fixed now to make sure that the salutation is persisted in the order database as well and is shown in Zed same way as in Yves. In addition, we update default list of salutations. Now customers are able to choose from Mr, Mrs, Ms and Dr.

Predis Constraint

In its last update PHP (7.0.18 / 7.1.4) introduced a bug in fsopen(). See PHP bug report here. In predis version 1.1.1 this problem is already fixed. We updated our constraints to pull this new patch version.

Image URL Length Constraint

URLs for product images were not allowed to be longer than 1024 chars. In some cases this could be too short. Considering that it’s better to use the HTTP spec limitations here as maximum length, URLs for a product images are now allowed to be up to 2048 instead of 1024 characters long.
